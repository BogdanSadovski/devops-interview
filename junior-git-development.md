# Junior DevOps — Git & Development

## Git: базовые понятия

**Git** — распределённая система контроля версий, позволяющая хранить историю изменений кода и работать в команде.

Основные сущности:

* repository — хранилище кода
* commit — зафиксированное изменение
* branch — ветка разработки
* HEAD — указатель на текущий коммит

---

## Git pull vs Git fetch

* **git fetch** — забирает изменения с удалённого репозитория, но не применяет их
* **git pull** = fetch + merge (или rebase)

Использовать fetch безопаснее, чтобы сначала посмотреть изменения.

---

## Merge conflict

Возникает, когда Git не может автоматически объединить изменения.

Решение:

1. Открыть конфликтные файлы
2. Выбрать нужный вариант кода
3. Удалить conflict markers
4. Сделать commit

---

## Git merge vs Git rebase

* **merge** — сохраняет историю, создаёт merge commit
* **rebase** — переписывает историю, делает её линейной

Для shared веток безопаснее merge.

---

## Cherry-pick

Позволяет применить конкретный коммит из другой ветки:

```bash
git cherry-pick <commit>
```

---

## Git-flow

Популярная модель ветвления:

* main/master — production
* develop — основная ветка разработки
* feature/* — новые фичи
* release/* — подготовка релиза
* hotfix/* — срочные исправления

---

## GitHub vs GitLab vs Bitbucket

* GitHub — популярный, open source комьюнити
* GitLab — встроенный CI/CD
* Bitbucket — интеграция с Jira

---

## Версионирование

### SemVer

Формат: MAJOR.MINOR.PATCH

* MAJOR — breaking changes
* MINOR — новая функциональность
* PATCH — багфиксы

### CalVer

Основан на дате: YYYY.MM

---

## Компилируемые vs интерпретируемые языки

* Компилируемые: Go, C++, Rust
* Интерпретируемые: Python, Bash

Компилируемые быстрее, интерпретируемые проще в разработке.

---

## TDD

**Test Driven Development**:

1. Пишем тест
2. Пишем код
3. Рефакторим

Повышает качество и стабильность кода.

---

## Роль DevOps в разработке

DevOps отвечает за:

* автоматизацию сборки
* стандарты репозиториев
* CI/CD пайплайны
* безопасность и контроль версий
