# DevOps Junior — Networks

## 1. Что такое OSI и TCP/IP?

**OSI** — эталонная модель из 7 уровней, описывающая, как данные передаются по сети.

**TCP/IP** — практическая модель, используемая в реальных сетях.

### Соответствие уровней:

* Application (OSI 7–5)
* Transport (OSI 4)
* Internet (OSI 3)
* Network Access (OSI 2–1)

---

## 2. Зачем нужны network masks?

Маска сети определяет:

* какая часть IP — сеть
* какая часть — хост

Пример:

* `192.168.1.10/24`
* сеть: `192.168.1.0`
* хостов: 254

---

## 3. Структура IP-пакета и фрагментация

IP-пакет состоит из:

* заголовка (IP, TTL, protocol)
* полезной нагрузки

### Фрагментация

Происходит, если пакет больше MTU канала.

---

## 4. Что такое коллизия?

Коллизия — одновременная передача данных несколькими устройствами.

Актуально для:

* half-duplex
* старых Ethernet-сетей

В современных switched-сетях почти отсутствует.

---

## 5. Что такое proxy?

Proxy — промежуточный сервер между клиентом и целевым ресурсом.

### Используется для:

* кэширования
* безопасности
* логирования

---

## 6. Что такое firewall?

Firewall — система фильтрации сетевого трафика.

### Виды:

* stateless
* stateful
* network / host-based

---

## 7. Что такое NAT и зачем он нужен?

NAT преобразует private IP в public.

### Зачем:

* экономия IPv4
* скрытие внутренней сети

---

## 8. Типы IP-адресов

* Public / Private
* IPv4 / IPv6
* Static / Dynamic
* Loopback (`127.0.0.1`)

---

## 9. Ping и Traceroute

### Ping

* протокол: ICMP
* порт: отсутствует

### Traceroute

* ICMP / UDP / TCP (реализация зависит от ОС)

---

## 10. Что такое TCP?

TCP — надёжный протокол передачи данных.

Особенности:

* установка соединения
* контроль доставки
* порядок пакетов

---

## 11. TCP vs UDP

| TCP       | UDP            |
| --------- | -------------- |
| Надёжный  | Быстрый        |
| Сессия    | Без сессии     |
| HTTP, SSH | DNS, streaming |

---

## 12. Что такое порт?

Порт — логическая точка доступа к сервису.

Примеры:

* 22 — SSH
* 80 — HTTP
* 443 — HTTPS

---

## 13. Что такое DNS?

DNS сопоставляет:

* имя хоста
* IP-адрес

### Основные записи:

* A
* AAAA
* CNAME

---

## 14. Что такое socket?

Socket = IP + Port + Protocol

Используется приложениями для сетевого взаимодействия.

---

## 15. Как проверить сетевые соединения?

```bash
ss -tuln
netstat -tuln
```

---

## 16. Как проверить доступность хоста?

```bash
ping host
traceroute host
```

---

## 17. Что такое default gateway?

Default gateway — маршрутизатор для выхода за пределы локальной сети.

---

## 18. Как работает DNS-резолвинг?

1. Проверка локального кэша
2. `/etc/hosts`
3. DNS-сервер
4. Root → TLD → Authoritative

---

## 19. Что такое VLAN?

VLAN — логическое разделение сети на уровне L2.

Используется для:

* безопасности
* сегментации

---

## 20. Как узнать IP-адрес в Linux?

```bash
ip a
hostname -I
```
