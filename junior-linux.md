# DevOps Junior — Linux

## 1. Архитектура операционной системы Linux

Linux состоит из следующих уровней:

1. **Hardware** — физическое оборудование
2. **Kernel (ядро)** — управляет CPU, памятью, устройствами, процессами
3. **System libraries** — интерфейс между ядром и приложениями (glibc)
4. **User space** — shell, сервисы, приложения

Ядро работает в **kernel space**, приложения — в **user space**.

---

## 2. Главное предназначение операционной системы

Операционная система:

* управляет ресурсами (CPU, RAM, disk, network)
* обеспечивает изоляцию процессов
* предоставляет API для приложений
* обеспечивает безопасность и стабильность

---

## 3. Зачем нужны файловые системы? Какие существуют?

Файловая система определяет, **как данные хранятся и читаются с диска**.

### Популярные FS:

* ext4 — стандарт Linux
* xfs — высокая производительность
* btrfs — snapshots
* tmpfs — в памяти

---

## 4. В чём разница между виртуализацией и контейнеризацией?

| Виртуализация    | Контейнеризация |
| ---------------- | --------------- |
| Своя ОС          | Общее ядро      |
| Медленнее        | Быстрее         |
| Большой overhead | Лёгкая          |

Контейнеры используют **namespaces** и **cgroups**.

---

## 5. Файловая структура Linux

* `/etc` — конфигурация
* `/dev` — устройства
* `/proc` — информация о процессах
* `/sys` — информация о ядре и устройствах
* `/lib` — системные библиотеки
* `/var` — логи, кэш, временные данные

---

## 6. Что такое Load Average?

Load Average показывает среднее количество процессов:

* выполняющихся
* ожидающих CPU
* ожидающих I/O

Показывается за 1 / 5 / 15 минут.

## 7. Soft link vs Hard link

| Hard link       | Soft link |
| --------------- | --------- |
| inode           | путь      |
| нельзя между FS | можно     |
| нельзя на dir   | можно     |

## 8. File permissions и зачем директориям +x

Права:

* r — read
* w — write
* x — execute

Для директории `+x` означает **право входа** в неё.

---

## 9. Zombie process

Zombie — завершившийся процесс, чей exit code не был считан родителем.

* не потребляет CPU
* занимает запись в таблице процессов

---

## 10. Инструменты мониторинга системы

* CPU: `top`, `htop`
* Memory: `free`, `vmstat`
* Disk: `df`, `du`, `iostat`
* Network: `ss`, `netstat`, `iftop`

---

## 11. Что такое swappiness?

Swappiness определяет, **как активно Linux использует swap**.

* 0 — минимальный swap
* 60 — default

Проверка:

```bash
cat /proc/sys/vm/swappiness
```

---

## 12. Как посмотреть свободное место на диске?

```bash
df -h
```

Размер каталогов:

```bash
du -sh /path
```

---

## 13. Что такое inode?

Inode — структура данных, содержащая:

* owner
* permissions
* timestamps
* pointers на блоки данных

Имя файла в inode **не хранится**.

## 15. Перенаправление вывода

```

## 17. Ctrl+C vs Ctrl+Z

* Ctrl+C — SIGINT
* Ctrl+Z — SIGTSTP

```

* SIGTERM (15)
* SIGKILL (9)
