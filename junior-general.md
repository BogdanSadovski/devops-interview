# DevOps Junior — Общие вопросы

## 1. Что такое DevOps?

**DevOps** — это культура, набор практик и инструментов, направленных на улучшение взаимодействия между разработкой (Dev) и эксплуатацией (Ops), с целью более быстрой, стабильной и предсказуемой доставки ПО в production.

### Ключевые идеи:

* совместная ответственность за продукт
* автоматизация (CI/CD, IaC)
* быстрое и безопасное внедрение изменений
* мониторинг и обратная связь

### Что хотят услышать на собеседовании:

* DevOps — это не должность, а подход
* цель — ускорить delivery без потери стабильности

---

## 2. Что происходит, когда вы вводите google.com в браузере?

Кратко по шагам:

1. **DNS-запрос** — браузер определяет IP-адрес `google.com`
2. **TCP-соединение** — трёхстороннее рукопожатие (SYN → SYN/ACK → ACK)
3. **TLS-handshake** — установка защищённого HTTPS-соединения
4. **HTTP-запрос** — браузер отправляет GET-запрос
5. **Ответ сервера** — HTML, CSS, JS
6. **Рендеринг страницы** — браузер строит DOM и отображает страницу

### Важно упомянуть:

* кеш DNS
* CDN
* HTTPS обязателен

---

## 3. Как работает HTTPS?

HTTPS = HTTP + TLS (SSL)

### Основные этапы:

1. Клиент запрашивает сервер
2. Сервер отправляет сертификат
3. Клиент проверяет сертификат через CA
4. Обмениваются ключами
5. Дальнейшее общение идёт в зашифрованном виде

### Зачем нужен:

* защита от MITM-атак
* целостность данных
* аутентификация сервера

---

## 4. Что такое Infrastructure as Code (IaC)?

**IaC** — это управление инфраструктурой с помощью кода.

### Примеры инструментов:

* Terraform
* CloudFormation
* Ansible

### Какие проблемы решает:

* ручные ошибки
* отсутствие воспроизводимости
* configuration drift

### Плюсы:

* версионирование
* повторяемость
* автоматизация

---

## 5. В чём разница между виртуализацией и контейнеризацией?

### Виртуализация:

* полноценная ОС
* гипервизор
* высокий overhead

### Контейнеризация:

* общее ядро
* изоляция через namespaces и cgroups
* быстрее и легче

### Что сказать на интервью:

> Контейнеры легче, быстрее запускаются и лучше подходят для микросервисов

---

## 6. В чём преимущества контейнеров?

* быстрый старт
* меньший размер
* одинаковая среда dev/prod
* удобство CI/CD

---

## 7. Что такое Load Average?

Load Average — это среднее количество процессов:

* выполняющихся
* ожидающих CPU
* ожидающих I/O

Обычно отображается за:

* 1 минуту
* 5 минут
* 15 минут

### Пример:

* LA = 1 на 1-core CPU → норма
* LA = 4 на 1-core CPU → перегрузка

## 8. В чём разница между hard и soft link?

| Hard link                        | Soft link         |
| -------------------------------- | ----------------- |
| Ссылается на inode               | Ссылается на путь |
| Не работает между FS             | Работает          |
| Не может ссылаться на директорию | Может             |

---

## 9. Что такое zombie process?

Zombie — процесс, который завершился, но его статус не был считан родительским процессом.

* не потребляет CPU
* занимает запись в process table

Решение — корректно завершить родительский процесс.

---

## 10. Что такое Bash-скрипт?

Bash-скрипт — это файл с набором команд shell, выполняемых последовательно.

### Пример:

```bash
#!/bin/bash
echo "Hello, DevOps"
```

Используется для автоматизации задач.

---

## 11. Ctrl+C vs Ctrl+Z

* **Ctrl+C** — отправляет SIGINT (прерывает процесс)
* **Ctrl+Z** — отправляет SIGTSTP (ставит на паузу)

---

## 12. Что делает grep?

`grep` ищет строки по шаблону.

```bash
grep "error" app.log
```

## 13. Что выведут команды?

```bash
echo hostname
```

Выведет строку `hostname`

```bash
echo $(hostname)
```

Выведет имя текущего хоста
